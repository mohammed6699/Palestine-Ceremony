# Free Palestine - Solidarity Web Application

## Project Overview

This project is a web application built to support the Palestinian cause. It serves as a platform for raising awareness, sharing information, and fostering a community of solidarity. The application aims to educate visitors about the history, culture, and ongoing struggles of the Palestinian people while providing a space to honor the memory of those who have been lost.

The key features include:
- An interactive historical timeline of key events.
- A memorial wall where users can leave messages of support and remembrance.
- Informational pages about Palestinian culture, stories, and awareness topics.
- A visually compelling gallery showcasing the beauty of Palestine's land and people.

## Tech Stack

This project was bootstrapped with [Vite](https://vitejs.dev/) and is built upon a modern front-end stack:

- **Framework:** [React](https://reactjs.org/)
- **Build Tool:** [Vite](https://vitejs.dev/)
- **Styling:** [Tailwind CSS](https://tailwindcss.com/)
- **State Management:** [Redux Toolkit](https://redux-toolkit.js.org/)
- **Routing:** [React Router](https://reactrouter.com/)
- **HTTP Client:** [Axios](https://axios-http.com/) for making API requests.
- **Linting:** [ESLint](https://eslint.org/) for code quality and consistency.

## Getting Started

### Prerequisites

- Node.js (v18.x or higher)
- npm or yarn

### Installation

1.  Clone the repository:
    ```sh
    git clone <repository-url>
    ```
2.  Navigate to the `client-side/free-palestine` directory:
    ```sh
    cd client-side/free-palestine
    ```
3.  Install the dependencies:
    ```sh
    npm install
    ```

### Running the Development Server

To run the application in development mode, execute the following command. This will start the Vite development server, typically on `http://localhost:5173`.

```sh
npm run dev
```

### Building for Production

To create a production-ready build of the application, run:

```sh
npm run build
```

This command bundles the application into static files in the `dist` directory, which can then be deployed to any static hosting service.
